后端常见错误及解决方法

By baicharming

发表于 2017-07-14

文章目录
  • 错误提示：ImportError: No module named ‘sqlalchemy’

    错误分析：系统没有“sqlalchemy”这个模块导致导入错误，“sqlalchemy”是Python3的第三方库，用PIP3安装即可。举一反三，如果提示ImportError: No module named ‘flask’，解决方法类似。
    解决方法：sudo pip3 install sqlalchemy

  • 错误提示：SyntaxError: invalid syntax

    错误分析：语法错误，往上看能找到提示语法错误的代码行，纠正其语法错误，此错误可以举一反三。
    解决方法：找到对应的代码，纠正语法错误。

  • 错误提示：TabError: inconsistent use of tabs and spaces in indentation

    错误分析：在缩进的地方使用tab键和space键不一致，在Python语法规范中，缩进必须是用tab键和space键中的一种，不能两者混合使用。
    解决方法：找到对应代码，统一使用tab键或者space键。

  • 错误提示：Access denied for user ‘root’@’localhost’ (using password: YES)

    错误分析：拒绝访问MySQL数据库，是因为与MySQL数据库连接代码中，写入的root账户密码错误。
    解决方法：应该在代码中正确写入MySQL的root账户密码。

  • 错误提示：Unknown database ‘Association’

    错误分析：未知的数据库“Association”，应该在数据库里面创建一个名为“Association”的数据库，此错误可以举一反三。
    解决方法：create database Association;

  • 错误提示：NameError: name ‘username’ is not defined

    错误分析：’username’ 没有被定义，是因为在使用“username”之前没有定义这个变量，或者没有导入这个变量等情况，此错误可以举一反三。
    解决方法：找到对应变量，在使用前进行定义。

  • 错误提示：AttributeError: ‘Userorml’ object has no attribute ‘getidbyname’

    错误分析：’Userorml’ 对象没有分配’getidbyname’这个方法导致分配错误，在调用方法前必须先定义，或者先导入，此错误可以举一反三。
    解决方法：找到未分配的方法，在调用之前进行定义或导入。

  • 错误提示：TypeError: getidbyname() missing 1 required positional argument: ‘username’

    错误分析：getidbyname()这个方法缺少一个’username’参数，应该补上对应参数，此错误可以举一反三。
    解决方法：在调用getidbyname()方法时应加上缺少的参数。

  • 错误提示：TypeError: a bytes-like object is required, not ‘str’

    错误分析：类型错误：需要一个字节的对象，而不是字符串，这一般是在跟MySQL数据库连接的时候出现的错误，MySQL数据库用的是gbk编码，而项目数据库用的是utf-8编码。
    解决方法：修改orml文件夹下的dbbase.py文件中的 use_unicode，将 use_unicode=0改成 use_unicode=1，或者将 use_unicode=0删除。

  • 错误提示：TypeError: b’Tom’ is not JSON serializable

    错误分析：通过sqlalchemy查询出来的数据Tom是字节类型，而不是JSON序列，需要将其解码成utf-8类型。
    解决方法： b’Tom’ .decode(“utf-8”)

  • 错误提示：OSError: [Errno 99] Cannot assign requested address

    错误分析：不能分配请求地址，是因为在后端全局配置文件中，服务IP地址不是本机地址，应填写本机IP地址。
    解决方法：通过ifconfig命令查看本机IP地址，将其写入后端服务IP地址。
